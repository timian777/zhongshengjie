## §0 背景与范围说明

### §0.1 M2-β 遗留问题溯源

M2-β 把 `.vectorstore/core/` 下的 Python 框架（workflow.py / *_api.py / config_loader.py / world_config_loader.py 等）整体迁移到 `core/` 包，并把旧实现归档到 `.archived/vectorstore_core_20260418/`。但**胶水层没清理干净**，目前还有 8 个 .py 文件继续做 `sys.path.insert(0, ".vectorstore/core")` 然后 `from config_loader import ...`，运行时变成静默 fallback（HAS_CONFIG_LOADER=False），功能降级但不报错——所以测试 450 passed 也掩盖不了这个债务。

**真正的修法**：现在 `core/config_loader.py` 已存在并稳定，所有 `from config_loader import X` 应改成 `from core.config_loader import X`，同时删除 `.vectorstore/core` 那段 sys.path 注入。

## §4 N16 — 修 N10 lambda 签名 bug（链路 B 阻塞器）

### §4.1 问题定位

文件: [core/conversation/missing_info_detector.py](core/conversation/missing_info_detector.py)

**调用方** (line 195-203):

```python
if intent == "start_chapter":
```

