# 写手工作流启动手册

> 版本: v1
> 创建时间: 2026-04-18 22:05 (Asia/Shanghai)
> 适用范围: M2-β 重构后（M7 收口）

## 概述

M2-β 之后，旧的 `python -m core create --workflow` CLI 入口已停用。
小说创作工作流统一通过 **Claude Code skill** 启动。

## 启动方式

### 方式 1：完整工作流（推荐）

在 Claude Code 中输入：

```
/novel-workflow
```

这会触发 `novel-workflow` skill，自动：
1. 检索知识库（小说设定 / 创作技法 / 案例）
2. 分配主责写手
3. 生成初稿
4. 鉴赏师评分
5. 迭代修订（最多 N 次）

### 方式 2：单写手 skill

| Skill | 角色 | 擅长场景 |
|---|---|---|
| `/novelist-canglan` | 苍澜 | 玄幻、修真、宏大叙事 |
| `/novelist-xuanyi` | 玄一 | 古风、诗词、隐喻 |
| `/novelist-moyan` | 墨言 | 言情、心理、抒情 |
| `/novelist-jianchen` | 剑尘 | 武侠、动作、战斗 |
| `/novelist-yunxi` | 云溪 | 治愈、日常、烟火气 |

### 方式 3：自然对话

直接在 Claude Code 对话框里说：

```
请帮我写第3章的战斗场景
```

Claude 会自动选择合适的 skill 调用。

## 已知限制

- **知识库工具层断链**：M2-β 后写手 skill 的 `from .vectorstore.core.worldview_api import` 等
  调用全部失效。当前写手只能依赖**身份层**（人设、风格、词汇偏好）创作，
  无法直接检索向量库。如需检索，请显式调用：
  - `python -m core kb --search-novel "查询词"`
  - `python -m core kb --search-technique "查询词"`
  - `python -m core kb --search-case "查询词"`
  把检索结果手动喂给写手 skill。

- **链路 C 自动化已移除**：M2-β 之后没有自动 evaluator-revisor 闭环 CLI，
  迭代修订需要在对话里手动说 "请鉴赏师评分" + "按建议修订"。

## 知识库状态确认

启动前可跑：

```bash
python -m core kb --stats
```

应输出：
```
小说设定库: 160
创作技法库: 986
案例库: 387377
```

如数字异常请先：

```bash
python -m core kb --sync
```

## 故障排查

### Q: skill 报 `ModuleNotFoundError: modules.creation`

A: 旧 skill 版本。M7 之后 novel-workflow skill 的 4 处 `from modules.creation import`
   应该已被 stub 化（如果未修复，请联系维护者重新打包 skill）。

### Q: skill 报 `.vectorstore/core/worldview_api.py 不存在`

A: 同上，写手 skill 的 6 处硬编码路径需要清理。临时绕过：让写手放弃工具层调用，
   只走身份层创作。

### Q: `python -m core create --workflow` 输出迁移提示

A: 这是 N13 修复后的预期行为。请改用 skill 入口（见上文方式 1-3）。
