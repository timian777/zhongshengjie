## Task 2：解决合并冲突（仅当 Task 1 出现 CONFLICT 时执行）

### 2-1 查看冲突文件列表
```bash
git status --short | grep "^UU\|^AA\|^DD"
```

### 2-2 对 README.md 的冲突：取 master 版本为基础

```bash
# 取 master 的 README.md（保留公开版本号体系和 v0.1.0-preview changelog）
git checkout --theirs README.md
git add README.md
```

**原因**：master 的 README 已有 v0.1.0-preview changelog，公开版本号体系完整；
v2-dev 的 README 用内部版本号（v14.0）；合并后用 master 版本再在 Task 3 叠加 v0.2.0 更新。

### 2-3 对其他冲突文件（如 docs/ 下的文件）

```bash
# 逐一查看冲突文件，原则：取 v2-dev 版本（更新）
git checkout --theirs <冲突文件路径>
git add <冲突文件路径>
```

### 2-4 完成冲突解决，创建合并提交

```bash
git merge --continue
# 编辑器会弹出，直接保存退出（使用 Task 1 中已预写的 commit message）
```

如果 `--continue` 报错（no editor），使用：
```bash
GIT_EDITOR=true git merge --continue
```
