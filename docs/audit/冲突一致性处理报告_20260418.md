## 1. 目标与范围
- 统一权威：设定是规则唯一真源；大纲用于剧情与上下文注入；技法为方法论库。
- 发现与修复：自动发现“大纲 vs 设定”的重复、冲突、缺口，并将缺口转为设定 PATCH，按 5.4 规范完成入库与验证。
- 非功能性要求：
  - 不改动业务逻辑，不破坏既有测试与功能路径。
  - 工具可复用到其他作品（去除“血牙”等硬编码）。

## 2. 触发与对话用法
- 扔素材 → 自动编排：novel-inspiration-ingest skill（Use IMMEDIATELY）。
- 审计与整理：
  - “整理下大纲和设定” → 自动执行 全量审计→生成 PATCH→落盘→图谱更新→向量库同步→检索验证→落档。
  - “做冲突检查：大纲 vs 设定” → 输出冲突清单（含 {文件:行号}）+ 修复建议。
- 设定变更（对话指令）→ 路由 setting_update → 生成可读改动预览 → 确认后落盘与入库。

## 3. 实施架构与数据流
```
大纲（章节/总） --(ChapterOutlineParser:key_settings)--> 审计脚本
                               ↓                         ↘
设定（知识图谱 JSON 展平：点分字段） <-------------------- 对照/分类（duplicate/conflict/outline_only）
                               ↓
outline_only → 生成设定 PATCH（可审）→ 落盘 设定/*.md → 同步图谱（knowledge_graph.json） →
sync_manager --target novel → novel_settings（Qdrant） → UnifiedRetrievalAPI 检索验证 → 落档
```

## 4. 关键实现（文件与要点）
- 审计脚本（只读）：tools/audit_outline_vs_settings.py
  - 解析源：章节大纲/*.md 与 总大纲.md（key_settings 表）；展平图谱并支持“点分字段”（如 过往经历.年龄时间线）。
  - 实体解析：动态读取图谱实体名，做“最长前缀匹配”解析“合并键”（实体名+字段名）。
  - 字段别名：从 config/audit_field_map.json 读取 field_aliases；若无配置，使用通用别名（年龄/父亲之死/幸存原因/目睹内容/仇恨指向→过往经历.*）。
  - 输出：.cache/outline_vs_settings_report.json（duplicates/conflicts/outline_only/样例）。
- 入库规范（严格执行 5.4）：
  - 更新 .vectorstore/knowledge_graph.json（同步新增/细化字段）。
  - python -m modules.knowledge_base.sync_manager --target novel → --status。
  - 验证：point 增量 > 0，检索命中可见。
- 文档归档：docs/audit/README.md（说明、样例、命令、修复建议）。
