# 第一部分：安装（只做一次，约 2 小时）

## 步骤 1：安装 Python

**去哪里下载：** `https://www.python.org/downloads/`

选 **Python 3.12.x**，点击下载 Windows installer (64-bit)。

**安装时最重要的一步：**

> ⚠️ 打开安装程序后，在最底部看到 **"Add python.exe to PATH"** 选项，**必须打勾！**
> 然后点 "Install Now"。

**验证是否成功：**

按 `Win + R`，输入 `cmd`，回车。在黑色窗口里输入：

```
python --version
```

看到 `Python 3.12.x` → 成功。

---

## 步骤 2：安装 Git

**去哪里下载：** `https://git-scm.com/download/win`

下载 64-bit 版本，双击安装，一路点 **Next**，最后点 Finish。

**验证：**

```
git --version
```

看到版本号 → 成功。

---

## 步骤 3：安装 Docker Desktop

Docker 用来运行数据库，是必须安装的。

**去哪里下载：** `https://www.docker.com/products/docker-desktop/`

点 **Download for Windows**，下载完双击安装。

> 安装过程中可能提示安装 WSL2（Windows 的 Linux 子系统），点"安装"或"继续"。
> 安装完成后**必须重启电脑**。

重启后桌面右下角会出现鲸鱼图标 🐳，说明 Docker 在运行。

**验证：**

```
docker --version
```

看到版本号 → 成功。

---

## 步骤 4：下载项目文件

打开命令提示符，输入以下命令（一行一行执行）：

```
d:
mkdir novels
cd novels
git clone https://github.com/[老师提供的地址]
cd 众生界
```

看到 `core/`、`modules/`、`tools/` 等文件夹 → 成功。

---

## 步骤 5：安装依赖和 AI 模型

**5.1 安装依赖包**（在项目目录内执行）

```
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

大约 5-15 分钟，看到 `Successfully installed` → 成功。

**5.2 下载 AI 语言模型（BGE-M3，约 2.3 GB）**

```
set HF_ENDPOINT=https://hf-mirror.com
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('BAAI/bge-m3'); print('模型下载完成')"
```

看到 **模型下载完成** → 成功（需要 10-30 分钟）。

> 如果老师提供了离线模型文件，见文末附录的离线配置说明。

---

## 步骤 6：启动 Qdrant 数据库

Qdrant 是用来存储所有小说知识的数据库。

**6.1 拉取 Qdrant 镜像**（只做一次）

```
docker pull qdrant/qdrant
```

**6.2 创建数据存放目录**（只做一次）

```
mkdir D:\qdrant_data
```

**6.3 启动 Qdrant 容器**（只做一次，之后自动启动）

```
docker run -d --name qdrant --restart unless-stopped -p 6333:6333 -p 6334:6334 -v D:\qdrant_data:/qdrant/storage qdrant/qdrant
```

**验证：**

```
docker ps
```

看到 qdrant 容器状态是 `Up` → 成功。

也可以打开浏览器访问 `http://localhost:6333/dashboard`，看到界面 → 成功。

---

## 步骤 7：安装 AI 对话工具

本系统支持两种工具，建议都装：

| 工具 | 适合做什么 | 安装地址 |
|------|-----------|---------|
| Claude Code | 日常创作、世界观建立、写章节 | `https://claude.ai/code` |
| opencode | 批量任务（提炼整本小说、大量同步） | `https://github.com/sst/opencode/releases` |

**Claude Code 配置 API Key：**

```
claude config set api-key sk-ant-api03-你的key
```

**opencode 配置 API Key：**

```
opencode config set anthropic-api-key sk-ant-api03-你的key
```

> API Key 由老师统一提供，妥善保管，不要告诉别人。

---

## 步骤 8：安装技能包

技能包是告诉 AI 如何充当各种角色的说明书（5 位写手 + 多个专家角色）。

**8.1 创建技能目录**

```
mkdir C:\Users\%USERNAME%\.agents\skills
```

**8.2 安装**

老师会提供 ZIP 文件（`众生界技能包.zip`）。解压后，把所有文件夹复制到：

```
C:\Users\你的用户名\.agents\skills\
```

> 不知道你的用户名？在命令提示符里输入 `echo %USERNAME%` 就能看到。

**验证：**

```
dir C:\Users\%USERNAME%\.agents\skills\
```

看到 `novelist-canglan`、`novel-workflow` 等文件夹 → 成功。

# 第二部分：创建你的第一本小说（约 30 分钟）

## 步骤 9：复制配置文件并修改

在项目目录里执行：

```
copy config.example.json config.json
```

用**记事本**打开 `config.json`，只需修改以下 3 处：

```json
"worldview": {
    "current_world": "天道余烬",    ← ① 改成你的小说名
    ...
},
"validation": {
    "realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"],  ← ② 改成你小说的境界体系
    ...
},
"model": {
    "model_path": null,             ← ③ 离线模型时才需要改，否则保持 null
    ...
}
```

> 详细说明见文末附录。

---

## 步骤 10：初始化数据库存储空间

这一步创建系统所需的所有 Qdrant 集合（只做一次）：

```
python tools/data_builder.py --init
```

看到 `初始化完成` 或 `collections created` → 成功。

---

## 步骤 11：启动 Claude Code，创建你的小说

进入项目目录，启动对话：

```
d:
cd novels\众生界
claude
```

进入对话界面后，输入：

```
请根据我的描述帮我建立世界观配置，我的小说叫"天道余烬"，玄幻类型。

设定：修炼时代末期，天道法则开始崩溃，主角是普通少年，觉醒了"天道之眼"血脉。
力量体系：修仙，五个境界（觉醒期→凝气期→通脉期→融道期→天道期）
代价：使用天道之眼会消耗记忆，用得越多越忘记重要的人
三大势力：正道联盟、魔道联盟、异人组织
```

AI 会引导你逐步完善。描述完后说：

```
世界观信息我说得差不多了，请生成配置文件并同步到知识库
```

系统自动保存配置，后续创作时写手会自动加载。

# 第三部分：写你的第一章

## 每次使用前的检查清单

每次开机后，先确认 Docker 和 Qdrant 在运行：

```
docker ps
```

看到 qdrant 状态是 `Up` → 可以开始。如果没看到：

```
docker start qdrant
```

然后进入项目目录启动 Claude Code：

```
d:
cd novels\众生界
claude
```

## 写章节

在 Claude Code 里输入：

```
使用 novel-workflow，写第一章。
这章的重点是主角在一场意外中觉醒天道之眼，
我想体现觉醒有代价：他忘记了母亲的名字。
```

**创作过程中你会参与的关键环节：**

**① 阶段0：需求澄清**

AI 会问你一些问题来细化方向，认真回答，这会影响整章质量。

**② 阶段5.5：创意协商**（最重要）

AI 写完初稿后，鉴赏师会给出改进建议，你来决定接受哪些：

```
AI：【鉴赏师建议】
    1. 觉醒时刻建议用旁观者视角而非主角主观视角，增加震撼感
    2. 失忆的瞬间可以插入一个日常细节（比如忘了母亲惯用的口头禅），更克制

你：第1条不要，我需要主角自己的视角。第2条很好，采用。
```

**③ 输出结果**

最终章节文件保存在 `正文/` 目录。

# 第四部分：让系统越来越懂你

> 这部分**不需要一次性完成**，边写边补效果最好。

## 你需要逐步建立的4类内容

| 类型 | 存在哪里 | 作用 |
|------|---------|------|
| 大纲 | `总大纲.md` + `章节大纲/` | 写手创作时的故事地图 |
| 设定 | `config/worlds/你的小说.json` | 世界规则、人物、势力 |
| 技法 | `创作技法/`（11个维度分类） | 写手参考的写法库 |
| 评估标准 | 品味记忆（通过对话建立） | 让评估师符合你的风格 |

**核心原则：你只需要说话，系统自动判断是哪类内容并存入对应位置。**

---

## 4.1 建立大纲

**建立总大纲**（第一次写作前）：

```
帮我建立"天道余烬"的总大纲：
主线：觉醒天道之眼→被追杀流亡→融入异人组织→揭开天道崩溃真相
全书约30章，帮我规划各阶段核心事件
```

**写某章前建立章节大纲**：

```
使用 novel-workflow，帮我规划第三章的大纲：
主角第一次进入虚空裂缝，遭遇古神残影，3个场景，每场有结尾钩子
```

**更新大纲**（写作中发现新方向）：

```
我想在第15章加一个反转：正道联盟盟主是幕后主谋，
请帮我在第12章埋下伏笔，修改总大纲
```

---

## 4.2 持续完善设定

直接说出你的想法，系统自动识别并存入设定：

```
我的小说里新增了一个势力"虚空商会"，中立商业组织，
跨越各大阵营，任何人给钱都服务
```

```
正道联盟盟主"苍云子"，70岁，修为天道期，
外表仙风道骨，实则心机深沉
```

```
天道之眼的代价需要修改：不是消耗真气，
改成消耗记忆，用多了会忘记重要的人
```

---

## 4.3 向技法库学习写法

**从你欣赏的内容中学（随时可做）：**

```
使用 novel-inspiration-ingest，提炼以下片段，存入"情感维度"：

[粘贴 100-500 字的内容]

这段的亮点是：父亲弄错孩子名字这个细节，写出了多年的疏离感
```

**检索技法库找灵感：**

```
使用 novelist-technique-search，我要写一个失败场景，
帮我找"战斗冲突维度"和"情感维度"下适合的技法
```

---

## 4.4 告诉评估师你的风格

```
告诉评估师我的写作风格：慢节奏，注重代价感，
情感通过细节传达而不是直白表述
```

```
告诉评估师：我喜欢用短句独立成段制造停顿感，不要判为失败
```

推翻不认同的评估：

```
我不同意这个评估，第三段的快节奏是我故意为之，
我要那种窒息感，记录：我认为这段通过
```

# 常见问题

**Q：运行命令出现乱码**

```
chcp 65001
```

执行这行后再运行其他命令。

---

**Q：pip install 一直失败**

- 确认 Python 版本是 3.12.x（`python --version`）
- 加国内镜像：命令里已有 `-i https://pypi.tuna.tsinghua.edu.cn/simple`
- 权限问题：右键命令提示符，选"以管理员身份运行"

---

**Q：Docker Desktop 打不开**

1. 确认 Windows 版本（开始菜单搜"winver"），需要是 2004 或更高
2. BIOS 中需要开启虚拟化（一般默认开启）
3. 尝试重装 Docker Desktop

---

**Q：Qdrant 容器没有启动**

```
docker ps -a
```

- 看到 qdrant 状态是 `Exited` → `docker start qdrant`
- 完全没有 qdrant 容器 → 重新执行步骤 6 的 `docker run` 命令
- 看报错原因 → `docker logs qdrant`

---

**Q：AI 说"找不到世界观配置"**

```
帮我检查当前小说"天道余烬"的世界观配置是否正常
```

如果丢失，重新描述世界观，通过对话重建即可，不影响已有章节。

---

**Q：BGE-M3 模型下载太慢或失败**

1. 确认设置了镜像：`set HF_ENDPOINT=https://hf-mirror.com`
2. 实在不行，找老师要离线模型文件，按附录配置

---

**Q：AI 生成内容感觉很普通**

初期正常，系统还不了解你的品味：

1. 明确告诉 AI 不喜欢什么："这个太普通了，换个思路"
2. 用 `novel-inspiration-ingest` 录入你欣赏的写法
3. 积累 3-5 章后明显改善

---

**Q：我想写另一本小说**

```
使用 novel-workflow，切换到我的另一本小说"星海纪元"
```

如果"星海纪元"还没有建立，系统会引导你创建。

# 附录：config.json 只需改这 3 处

打开 `config.json`（记事本即可），找到并修改以下内容。其他内容**不要动**。

---

### ① current_world（必须改）

```json
"worldview": {
    "current_world": "天道余烬"
```

改成**你的小说名**（和你告诉系统的名字完全一致）。

---

### ② realm_order（必须改）

```json
"validation": {
    "realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"]
```

改成**你小说的境界体系**，从低到高排列。系统用这个来检测"境界是否合法提升"。

如果你的小说不需要境界检测，留空数组即可：`"realm_order": []`

---

### ③ model_path（只有离线时才改）

```json
"model": {
    "model_path": null
```

如果老师给了离线模型文件，改成模型文件夹的路径：

```json
"model_path": "D:/models/bge-m3"
```

注意路径用正斜杠 `/` 而不是反斜杠 `\`。

---

### 其他常用配置（通常不需要改）

| 配置项 | 默认值 | 什么时候改 |
|--------|--------|-----------|
| `qdrant_host` | `"localhost"` | 换成云端 Qdrant 时改 |
| `qdrant_port` | `6333` | 改了 Docker 端口映射时才改 |
| `auto_sync` | `true` | 保持 true，设定更新后自动同步 |
| `skip_rules` | `[]` | 某条校验规则总是误报时，把规则编号加进去 |
