# 第一部分：安装（用 AI 自动完成，约 1-2 小时）

## 第 0 步：先安装 opencode（只做一次）

opencode 是驱动自动安装的 AI 工具。

1. 去 `https://github.com/sst/opencode/releases` 下载最新的 Windows 版本（文件名类似 `opencode_windows_x64.zip`）
2. 解压后把 `opencode.exe` 放到任意目录，比如 `D:\tools\`
3. 打开命令提示符（`Win + R` → 输入 `cmd` → 回车），输入：

```
D:\tools\opencode.exe
```

看到 opencode 的对话界面 → 成功。

> 如果老师已经帮你安装好了 opencode，直接从第 1 步开始。

opencode 会自动逐步完成安装，遇到需要你填写信息时（比如小说名、API Key、资源路径）会停下来用中文问你。

# 第三部分：让系统越来越懂你

> 这部分**不需要一次性完成**，边写边补效果最好。

## 你需要逐步建立的4类内容

| 类型 | 存在哪里 | 作用 |
|------|---------|------|
| 大纲 | `总大纲.md` + `章节大纲/` | 写手创作时的故事地图 |
| 设定 | `config/worlds/你的小说.json` | 世界规则、人物、势力 |
| 技法 | `创作技法/`（11个维度分类） | 写手参考的写法库 |
| 评估标准 | 品味记忆（通过对话建立） | 让评估师符合你的风格 |

**核心原则：你只需要说话，系统自动判断是哪类内容并存入对应位置。**

# 常见问题

**Q：运行命令出现乱码**

```
chcp 65001
```

执行这行后再运行其他命令。

---

**Q：pip install 一直失败**

- 确认 Python 版本是 3.12.x（`python --version`）
- 加国内镜像：命令里已有 `-i https://pypi.tuna.tsinghua.edu.cn/simple`
- 权限问题：右键命令提示符，选"以管理员身份运行"

---

**Q：Docker Desktop 打不开**

1. 确认 Windows 版本（开始菜单搜"winver"），需要是 2004 或更高
2. BIOS 中需要开启虚拟化（一般默认开启）
3. 尝试重装 Docker Desktop

---

**Q：Qdrant 容器没有启动**

```
docker ps -a
```

- 看到 qdrant 状态是 `Exited` → `docker start qdrant`
- 完全没有 qdrant 容器 → 告诉 opencode "Qdrant 容器不见了"，让它重新执行启动命令
- 看报错原因 → `docker logs qdrant`

---

**Q：AI 说"找不到世界观配置"**

```
帮我检查当前小说"天道余烬"的世界观配置是否正常
```

如果丢失，重新描述世界观，通过对话重建即可，不影响已有章节。

---

**Q：BGE-M3 模型下载太慢或失败**

1. 确认设置了镜像：`set HF_ENDPOINT=https://hf-mirror.com`
2. 实在不行，找老师要离线模型文件，按附录A配置

**Q：AI 生成内容感觉很普通**

初期正常，系统还不了解你的品味：

1. 明确告诉 AI 不喜欢什么："这个太普通了，换个思路"
2. 用 `novel-inspiration-ingest` 录入你欣赏的写法
3. 积累 3-5 章后明显改善

```
使用 novel-workflow，切换到我的另一本小说"星海纪元"
```

如果"星海纪元"还没有建立，系统会引导你创建。

# 附录A：config.json 配置说明

打开 `config.json`（记事本即可），找到并修改以下内容。其他内容**不要动**。

---

### ① current_world（必须改）

```json
"worldview": {
    "current_world": "天道余烬"
```

改成**你的小说名**（和你告诉系统的名字完全一致）。

---

### ② realm_order（必须改）

```json
"validation": {
    "realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"]
```

改成**你小说的境界体系**，从低到高排列。系统用这个来检测"境界是否合法提升"。

如果你的小说不需要境界检测，留空数组即可：`"realm_order": []`

---

### ③ model_path（只有离线时才改）

```json
"model": {
    "model_path": null
```

如果老师给了离线模型文件，改成模型文件夹的路径：

```json
"model_path": "D:/models/bge-m3"
```

注意路径用正斜杠 `/` 而不是反斜杠 `\`。

---

### 其他常用配置（通常不需要改）

| 配置项 | 默认值 | 什么时候改 |
|--------|--------|-----------|
| `qdrant_host` | `"localhost"` | 换成云端 Qdrant 时改 |
| `qdrant_port` | `6333` | 改了 Docker 端口映射时才改 |
| `auto_sync` | `true` | 保持 true，设定更新后自动同步 |
| `skip_rules` | `[]` | 某条校验规则总是误报时，把规则编号加进去 |

# 附录B：手动安装步骤（备用）

> 如果 opencode 无法使用，或你想了解安装的每一步具体做了什么，可以按这里的步骤手动完成。

---

## 步骤 1：安装 Python

**去哪里下载：** `https://www.python.org/downloads/`

选 **Python 3.12.x**，点击下载 Windows installer (64-bit)。

**安装时最重要的一步：**

> ⚠️ 打开安装程序后，在最底部看到 **"Add python.exe to PATH"** 选项，**必须打勾！**
> 然后点 "Install Now"。

**验证是否成功：**

按 `Win + R`，输入 `cmd`，回车。在黑色窗口里输入：

```
python --version
```

看到 `Python 3.12.x` → 成功。

---

## 步骤 2：安装 Git

**去哪里下载：** `https://git-scm.com/download/win`

下载 64-bit 版本，双击安装，一路点 **Next**，最后点 Finish。

**验证：**

```
git --version
```

看到版本号 → 成功。

---

## 步骤 3：安装 Docker Desktop

Docker 用来运行数据库，是必须安装的。

**去哪里下载：** `https://www.docker.com/products/docker-desktop/`

点 **Download for Windows**，下载完双击安装。

> 安装过程中可能提示安装 WSL2（Windows 的 Linux 子系统），点"安装"或"继续"。
> 安装完成后**必须重启电脑**。

重启后桌面右下角会出现鲸鱼图标 🐳，说明 Docker 在运行。

**验证：**

```
docker --version
```

看到版本号 → 成功。

---

## 步骤 4：下载项目文件

打开命令提示符，输入以下命令（一行一行执行）：

```
d:
mkdir novels
cd novels
git clone https://github.com/[老师提供的地址]
cd 众生界
```

看到 `core/`、`modules/`、`tools/` 等文件夹 → 成功。

---

## 步骤 5：安装依赖和 AI 模型

**5.1 安装依赖包**（在项目目录内执行）

```
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

大约 5-15 分钟，看到 `Successfully installed` → 成功。

**5.2 下载 AI 语言模型（BGE-M3，约 2.3 GB）**

```
set HF_ENDPOINT=https://hf-mirror.com
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('BAAI/bge-m3'); print('模型下载完成')"
```

看到 **模型下载完成** → 成功（需要 10-30 分钟）。

> 如果老师提供了离线模型文件，见附录A的离线配置说明。

---

## 步骤 6：启动 Qdrant 数据库

Qdrant 是用来存储所有小说知识的数据库。

**6.0 配置镜像加速**（国内网络必做，否则下一步会超时）

打开 Docker Desktop → 右上角齿轮图标 → **Docker Engine** → 在 JSON 里加入以下内容：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
```

点 **Apply & Restart**，等 Docker 重启完成（右下角鲸鱼图标变绿）。

> 如果老师提供了学校专用镜像地址，优先填学校的地址，效果更快。

**6.1 拉取 Qdrant 镜像**（只做一次）

```
docker pull qdrant/qdrant
```

> 如果拉取过程中卡住超过 3 分钟没有进度，按 `Ctrl+C` 停止，确认步骤 6.0 的镜像加速已生效后重试。

**6.2 创建数据存放目录**（只做一次）

```
mkdir D:\qdrant_data
```

**6.3 启动 Qdrant 容器**（只做一次，之后自动启动）

```
docker run -d --name qdrant --restart unless-stopped -p 6333:6333 -p 6334:6334 -v D:\qdrant_data:/qdrant/storage qdrant/qdrant
```

**验证：**

```
docker ps
```

看到 qdrant 容器状态是 `Up` → 成功。

也可以打开浏览器访问 `http://localhost:6333/dashboard`，看到界面 → 成功。

---

## 步骤 7：安装 AI 对话工具

本系统支持两种工具，建议都装：

| 工具 | 适合做什么 | 安装地址 |
|------|-----------|---------|
| Claude Code | 日常创作、世界观建立、写章节 | `https://claude.ai/code` |
| opencode | 批量任务（提炼整本小说、大量同步） | `https://github.com/sst/opencode/releases` |

**Claude Code 配置 API Key：**

```
claude config set api-key sk-ant-api03-你的key
```

**opencode 配置 API Key：**

```
opencode config set anthropic-api-key sk-ant-api03-你的key
```

> API Key 由老师统一提供，妥善保管，不要告诉别人。

## 步骤 8：安装技能包

技能包是告诉 AI 如何充当各种角色的说明书（5 位写手 + 多个专家角色）。

**8.1 创建技能目录**

```
mkdir C:\Users\%USERNAME%\.agents\skills
```

**8.2 安装**

```
C:\Users\你的用户名\.agents\skills\
```

> 不知道你的用户名？在命令提示符里输入 `echo %USERNAME%` 就能看到。

**验证：**

```
dir C:\Users\%USERNAME%\.agents\skills\
```

看到 `novelist-canglan`、`novel-workflow` 等文件夹 → 成功。

## 步骤 9：配置 config.json

在项目目录里执行：

```
copy config.example.json config.json
```

用**记事本**打开 `config.json`，只需修改以下 2 处（详见附录A）：

- `current_world`：改成你的小说名
- `realm_order`：改成你小说的境界体系
