## 2. 第一步：安装基础软件

### 2.1 安装 Python 3.12

**第一步：下载**

打开浏览器，访问：`https://www.python.org/downloads/`

找到 **Python 3.12.x**（x 是具体版本号，选最新的即可），点击下载 **Windows installer (64-bit)**。

**第二步：安装**

1. 双击下载的 `.exe` 文件
2. **重要：** 在安装界面底部，勾选 **"Add python.exe to PATH"**（这个必须勾！）
3. 点击 **"Install Now"**
4. 等待安装完成，点击 **"Close"**

**第三步：验证安装**

按 `Win + R`，输入 `cmd`，回车，打开黑色窗口（命令提示符）。

输入以下内容并回车：
```
python --version
```

如果看到 `Python 3.12.x`，说明安装成功。

---

### 2.2 安装 Git

**第一步：下载**

访问：`https://git-scm.com/download/win`

下载 **64-bit Git for Windows Setup**。

**第二步：安装**

双击安装文件，全程点 **"Next"**（默认设置即可），最后点 **"Finish"**。

**第三步：验证**

在命令提示符中输入：
```
git --version
```

看到 `git version x.x.x` 即为成功。

---

### 2.3 安装 Docker Desktop

Docker 是用来运行 Qdrant 数据库的容器平台，是本系统的**必须组件**。

**第一步：下载**

访问：`https://www.docker.com/products/docker-desktop/`

点击 **"Download for Windows"** 下载安装包（约 500MB）。

**第二步：安装**

1. 双击安装包，点击 **"OK"** 开始安装
2. 安装过程中可能提示安装 WSL2（Windows Subsystem for Linux），点击 **"安装"** 或 **"继续"**
3. 安装完成后，**重启电脑**

**第三步：启动并验证**

重启后，桌面上的 Docker Desktop 图标会自动启动（系统托盘区有一个鲸鱼图标 🐳）。

在命令提示符中验证：
```
docker --version
```

看到 `Docker version xx.xx.xx` 即为成功。

> ⚠️ **如果提示"WSL kernel update required"：** 访问 `https://aka.ms/wsl2kernel` 下载并安装内核更新，然后重启 Docker Desktop。

## 4. 第三步：安装项目依赖和 AI 模型

### 4.1 安装 Python 依赖包

在命令提示符中，确保你在项目目录内（路径显示 `D:\novels\众生界>`），执行：

```
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

大约需要 **5-15 分钟**。看到 **successfully installed** 就是成功了。

---

### 4.2 下载 AI 语言模型（BGE-M3）

系统使用 **BGE-M3**（BAAI 开发的中文理解模型）来分析和检索文字，约 **2.3 GB**。

#### 方法一：使用国内镜像自动下载（推荐）

在项目目录执行以下命令，设置国内镜像源后自动下载：

```
set HF_ENDPOINT=https://hf-mirror.com
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('BAAI/bge-m3'); print('模型下载完成')"
```

下载大约需要 10-30 分钟，看到 **模型下载完成** 即成功。

> 💡 **HuggingFace 国内镜像地址：** `https://hf-mirror.com`（由国内团队维护，速度快）

#### 方法二：老师提供离线模型文件

如果老师提供了模型文件包（通常是一个文件夹，含 `config.json`、`tokenizer.json` 等文件）：

1. 将模型文件夹解压到一个路径简单的位置，例如：
   ```
   D:\models\bge-m3\
   ```

2. 用**记事本**打开项目里的 `config.json`，找到 `"model"` 部分，修改 `"model_path"` 的值：

   ```json
   "model": {
       "embedding_model": "BAAI/bge-m3",
       "model_path": "D:/models/bge-m3",
       "hf_cache_dir": null,
       "vector_size": 1024,
       "batch_size": 20
   }
   ```

   将 `"model_path": null` 改为 `"model_path": "D:/models/bge-m3"`（填入你实际放置的路径，注意用正斜杠 `/`）。

3. 保存文件（Ctrl+S）。

#### 验证模型可用

```
python -c "from sentence_transformers import SentenceTransformer; m = SentenceTransformer('BAAI/bge-m3'); v = m.encode('测试'); print('模型正常，向量维度:', len(v))"
```

看到 `模型正常，向量维度: 1024` 即为成功。

## 5. 第四步：安装向量数据库 Qdrant（Docker 方式）

Qdrant 存储你所有的小说知识：世界观设定、写作技法、案例库、经验日志。本系统使用 **Docker 容器**运行 Qdrant，这是最稳定的方式。

### 5.1 拉取 Qdrant 镜像

确保 Docker Desktop 已启动（系统托盘有鲸鱼图标），然后在命令提示符中执行：

```
docker pull qdrant/qdrant
```

下载约 200 MB，完成后显示 `Status: Downloaded newer image for qdrant/qdrant:latest`。

> ⚠️ **如果拉取速度很慢：** 可以配置 Docker 镜像加速器。在 Docker Desktop 的 Settings → Docker Engine 中添加：
> ```json
> {
>   "registry-mirrors": ["https://mirror.ccs.tencentyun.com"]
> }
> ```
> 点击 Apply & Restart 后重新拉取。

### 5.2 创建数据存储目录

Qdrant 需要一个目录来存储数据（你的所有知识库内容都在这里）：

```
mkdir D:\qdrant_data
```

> 📌 **这个目录很重要，不要随意删除！** 你的所有知识库内容都存储在这里。

### 5.3 启动 Qdrant 容器

**每次使用系统前，都需要先启动 Qdrant。**

```
docker run -d --name qdrant --restart unless-stopped -p 6333:6333 -p 6334:6334 -v D:\qdrant_data:/qdrant/storage qdrant/qdrant
```

命令说明：
- `-d`：后台运行（不占用当前窗口）
- `--name qdrant`：容器命名为 qdrant，方便管理
- `--restart unless-stopped`：电脑重启后自动恢复
- `-p 6333:6333`：开放数据访问端口
- `-v D:\qdrant_data:/qdrant/storage`：把数据存到本地目录

**验证启动成功：**

```
docker ps
```

看到类似以下内容，说明 Qdrant 正在运行：
```
CONTAINER ID   IMAGE           STATUS          PORTS                    NAMES
abc123def456   qdrant/qdrant   Up 2 minutes    0.0.0.0:6333->6333/tcp   qdrant
```

也可以打开浏览器访问：`http://localhost:6333/dashboard`，看到 Qdrant 管理界面即为成功。

### 5.4 日常管理 Qdrant

| 操作 | 命令 |
|------|------|
| 启动 Qdrant | `docker start qdrant` |
| 停止 Qdrant | `docker stop qdrant` |
| 查看是否在运行 | `docker ps` |
| 查看日志（排错用） | `docker logs qdrant` |

> 💡 **好消息：** 设置了 `--restart unless-stopped` 后，电脑重启时 Qdrant 会自动恢复运行，只需确认 Docker Desktop 已启动即可，不需要每次手动启动容器。

## 6. 第五步：获取 AI 对话权限

本系统支持两种 AI 对话工具：**Claude Code**（主要）和 **opencode**（辅助）。两者都需要 API Key。

### 6.1 注册 Anthropic 账号并获取 API Key

访问 `https://console.anthropic.com/`，注册账号。

> 💡 如果老师提供了统一的 API Key，跳过注册步骤，直接使用老师给的 Key。

登录后，点击 **API Keys** → **Create Key**，复制这个 Key（形如 `sk-ant-api03-xxxxx`）。

**妥善保管这个 Key，不要告诉任何人。**

---

### 6.2 安装 Claude Code（主要对话界面）

Claude Code 是与 AI 进行创作对话的主要界面，适合日常章节创作、世界观建立、写手调用。

**安装：**

访问：`https://claude.ai/code`，按页面指引下载并安装（Windows 有一键安装包）。

安装完成后验证：
```
claude --version
```

**配置 API Key：**
```
claude config set api-key sk-ant-api03-你的key内容
```

**启动方式：** 在项目目录执行 `claude`，进入对话界面后，直接用中文对话即可。

---

### 6.3 安装 opencode（辅助执行工具）

opencode 是另一个 AI 终端，擅长执行批量任务、运行代码脚本、处理技术性操作（如批量提炼外部小说、执行知识库同步等）。两者搭配使用效果最好。

**安装方式一：通过 npm（需先安装 Node.js）**

1. 访问 `https://nodejs.org/`，下载并安装 **LTS 版本**的 Node.js
2. 安装完成后，在命令提示符中执行：
   ```
   npm install -g opencode-ai
   ```

**安装方式二：直接下载（推荐无编程基础的同学）**

访问：`https://github.com/sst/opencode/releases`，下载 Windows 版安装包（`.exe`）。

**配置 API Key：**
```
opencode config set anthropic-api-key sk-ant-api03-你的key内容
```

**验证：**
```
opencode --version
```

**启动方式：** 在项目目录执行 `opencode`，进入对话界面。

---

### 6.4 Claude Code vs opencode 用哪个？

| 场景 | 推荐工具 |
|------|---------|
| 日常创作（写章节、调写手、世界观建立） | Claude Code |
| 批量处理（提炼整本外部小说、批量同步） | opencode |
| 不确定 | Claude Code（更通用） |

> 💡 两个工具都在项目目录执行，功能基本相同，主要区别是 opencode 在处理长时间任务时不易超时。

## 8. 第七步：初始化你的小说项目

到这一步，系统安装完毕。现在通过**对话**来创建你自己的小说。

### 8.1 进入项目目录并启动 Claude Code

```
d:
cd novels\众生界
claude
```

进入 Claude Code 对话界面后，所有操作都通过对话完成。

### 8.2 运行初始化向导

在 Claude Code 对话中输入（把"我的玄幻世界"替换成你的小说名）：

```

```

### 8.3 通过对话建立你的世界观

世界观是你小说的基础。**不需要手动编辑任何文件**，通过对话告诉 AI 你的想法，系统帮你整理成结构化设定。

在 Claude Code 中输入：

```
使用 novelist-worldview-generator，帮我建立世界观。
我的小说叫"天道余烬"，是玄幻类型。
大概设定是：修炼时代末期，天道法则开始崩溃，
主角是一个普通少年，觉醒了被认为早已消失的"天道之眼"血脉。
主要有修仙和异能两种力量体系，三大势力：正道联盟、魔道联盟、异人组织。
```

AI 会引导你逐步完善：

```
AI：好的，我来帮你建立"天道余烬"的世界观。
    首先，关于修仙体系，你希望设置几个境界？
    比如：炼气→筑基→金丹→元婴→化神，或者你有自己的想法？

你：五个境界就好，名字我还没想好，你帮我起

AI：建议如下五个境界：觉醒期→凝气期→通脉期→融道期→天道期
    寓意从觉醒潜力到最终触碰天道法则的过程。你觉得如何？

你：很好，就用这个

AI：（继续引导设定势力、角色、核心主题……）
```

建立完成后，系统自动保存所有设定，后续写作时写手会自动加载这些内容。

## 10. 各模块使用指南

所有模块都通过 **Claude Code 或 opencode 对话**触发，无需手动执行代码。

### 10.1 主模块：章节创作工作流

**适用场景：** 写每一章的主要内容，从构思到输出完整章节。

**对话触发：**
```
使用 novel-workflow，写第一章
```
```
使用 novel-workflow，继续写第三章，这章重点是主角与反派的第一次正面交锋
```

**完整工作流程（系统自动完成，标 * 的需要你参与）：**

```
阶段0 *：需求澄清——AI 和你讨论这章的重点方向
    ↓
阶段1-3：自动检索世界观/人物/剧情素材
    ↓
阶段4：灵感引擎——AI 提出多个创意方向
    ↓
阶段5：鉴赏师审核创意质量
    ↓
阶段5.5 *：三方协商——你决定保留哪些创意
    ↓
阶段6：评估师检查成稿质量
    ↓
阶段7 *（如需）：你可以推翻评估，坚持自己的判断
    ↓
阶段8：经验写入——系统记录本次创作经验
    ↓
输出：完整章节文件，保存在 正文/ 目录
```

**三方协商阶段（你需要参与的最关键环节）：**

```
AI：鉴赏师提出以下创意建议：
    1. 觉醒时刻设计在母亲濒死场景，代价是忘记母亲的名字
    2. 加入一个旁观者视角，让觉醒更有仪式感
    3. 觉醒后立刻遭到追杀，节奏快速

你：保留第1条，必须要。第3条也好，但觉醒先结束再追杀。
    第2条不要，太刻意了。

AI：明白，以这个方向继续……
```

### 10.3 五位写手直接调用

你可以单独召唤某位写手来创作特定内容，不走完整工作流程。

| 写手 | 擅长领域 | 对话触发 |
|------|---------|---------|
| 苍澜 | 世界观架构、宏大设定、势力描写 | `使用 novelist-canglan` |
| 玄一 | 剧情伏笔、悬念设计、结构反转 | `使用 novelist-xuanyi` |
| 墨言 | 人物刻画、情感描写、心理独白 | `使用 novelist-moyan` |
| 剑尘 | 战斗场景、热血动作、力量展示 | `使用 novelist-jianchen` |
| 云溪 | 意境营造、诗意描写、章节润色 | `使用 novelist-yunxi` |

**示例对话：**

使用 novelist-jianchen，帮我写一段战斗场景：
主角第一次展现"天道之眼"的力量，对手是一个飞扬跋扈的贵族子弟，

```

```
使用 novelist-yunxi，把以下这段战斗描写润色一遍，
加强意境感，但不要改变情节内容：
[粘贴你要润色的内容]
```

### 10.5 外部小说提炼模块

**适用场景：** 你读到一本好书或好的网文，想把它的写作技法"学"进你的系统。

#### 方式一：提炼片段（直接对话）

```
使用 novel-inspiration-ingest，我想提炼这段文字的写作技法：

[粘贴你想学习的 200-500 字文字片段]

我觉得这段写得好的地方是：描写战斗时通过细节积累而不是宏观叙事来制造紧张感
```

系统会分析亮点，提取可复用的写法，存入你的技法库。

#### 方式二：提炼整本小说（opencode 批量处理）

如果你有一个 TXT 或 EPUB 格式的小说文件，可以让系统提炼全书技法：

**第一步：** 把小说文件放入项目目录下的 `.novel-extractor/` 文件夹。

**第二步：** 在 **opencode** 对话中执行（这种批量任务用 opencode 更稳定）：

```
我有一本小说 "斗破苍穹.txt" 放在 .novel-extractor/ 目录里，
请帮我提炼其中的写作技法并存入知识库。
重点关注：战斗节奏控制、人物成长弧线、伏笔设计。
```

opencode 会逐章分析，整个过程可能需要几十分钟到几个小时，你可以去做其他事情。

### 10.6 评估模块

**适用场景：** 对自己写好的内容进行质量评估，找出问题所在。

**对话触发：**
```
使用 novelist-evaluator，请评估以下内容：

[粘贴你的章节内容]
```

系统评估维度：
- **代价感：** 主角的胜利/能力是否有合理的代价
- **人物立体性：** 人物行为是否符合其设定，有没有矛盾
- **节奏控制：** 张弛是否合理，有没有过于平铺直叙
- **禁忌表达检测：** 识别"AI 味"表达（如"眼中闪过一丝"）、时间连接词滥用等

**推翻评估（坚持你的判断）：**

如果评估结果你不认同：
```
我不同意这个评估，第三段的快节奏是我故意为之，
我想要那种窒息感，不是节奏问题
```

系统会记录你的判断，作为品味记忆，影响未来的评估标准。

## 11. 构建你的小说体系

### 11.1 建立世界观的正确顺序

**不要在创作前把所有设定都想好**，这样会陷入完美主义的陷阱。推荐的方式是：

```
第一阶段：建立基础（第1-2天）
├── 通过对话建立核心世界观：力量体系、1-2个主要势力、主角设定
├── 用 novel-workflow 写第一章试水
└── 不用想太多，先写，写的过程中自然会完善设定

第二阶段：丰富知识库（第3-7天）
├── 把你欣赏的写作片段通过对话喂给系统学习
├── 写到哪里，设定补充到哪里（通过对话告诉 worldview-generator）
└── 每写完一章，系统自动积累经验

第三阶段：进入正循环（第二周起）
├── 每次创作 → 系统学习经验 → 下次提案更准
├── 你调整创意 → 系统记录你的品味 → 越来越懂你
└── 知识库越丰富，写手参考的素材越多，质量越高
```

### 11.2 你的品味如何塑造系统

系统有一个"记忆点库"，记录你在创作过程中**喜欢**和**不喜欢**的表达。越用越懂你。

**主动告诉系统你的品味：**

```
这段写得很好，我喜欢这种"在行动中展现人物性格"的写法，记住这个风格
```

```
这个创意不行，太俗套了，主角觉醒不要有特效光芒，我讨厌这种套路
```

### 11.3 世界观持续深化

随着你的创作，世界观会不断丰富。通过对话告诉系统：

```
使用 novelist-worldview-generator，我写到第五章时发现，
主角母亲其实是正道联盟的内奸，这个设定之前没有，帮我加进世界观
```

```
使用 novelist-worldview-generator，把"天道之眼"的代价补充一下：
每次使用会消耗一段记忆，用多了会忘记最重要的人
```

系统自动更新，后续写作立即生效。

### 11.4 章节经验的积累

每完成一章，系统自动记录：用了哪些技法、什么写法有效、什么失败了、给下一章的建议。

这些经验是你个人的创作数据，随着积累，系统对你写作习惯的理解会越来越深。

## 12. 常见问题

### Q1：运行命令出现乱码或中文显示问题

在命令提示符中先执行：
```
chcp 65001
```
然后再运行其他命令。

---

### Q2：pip install 一直报错

- **网络问题：** 命令中加上 `-i https://pypi.tuna.tsinghua.edu.cn/simple`
- **Python 版本不对：** 确认 `python --version` 显示 3.12.x
- **权限问题：** 右键命令提示符，选"以管理员身份运行"

---

### Q3：Docker Desktop 打不开或报错

1. 确认 Windows 版本是 2004 或更高（开始菜单搜索"winver"查看）
2. 确认 BIOS 中开启了虚拟化（一般默认开启，如不确定请询问老师）
3. 尝试重装 Docker Desktop

---

### Q4：Qdrant 容器没有启动

先查看是否已有容器：
```
docker ps -a
```

- 如果看到 qdrant 容器状态是 `Exited`：执行 `docker start qdrant`
- 如果完全没有 qdrant 容器：重新执行第5步的 `docker run` 命令
- 查看错误原因：`docker logs qdrant`

---

### Q5：对话中 AI 说"找不到世界观配置"

```
使用 novelist-worldview-generator，帮我检查一下当前小说的世界观配置是否正常，
我的小说名叫"我的玄幻世界"
```

如果配置丢失，通过对话重建即可，不会影响已有章节文件。

---

### Q6：BGE-M3 模型下载失败或速度太慢

1. 确认已设置镜像：`set HF_ENDPOINT=https://hf-mirror.com`
2. 如果网络实在不稳，联系老师获取离线模型文件
3. 配置离线路径见第 4.2 节"方法二"

---

### Q7：AI 生成的内容感觉很普通

这是正常的初期状态。系统需要了解你的品味：

1. 明确告诉 AI 不喜欢什么（"这个太普通了，换个角度"）
2. 用 `novel-inspiration-ingest` 录入你欣赏的写法
3. 积累 3-5 章后会有明显改善

---

### Q8：我想换一个小说来写

在 Claude Code 对话中：
```
使用 novel-workflow，切换到我的另一个小说"星海纪元"
```

如果"星海纪元"还没有建立，系统会引导你创建。
