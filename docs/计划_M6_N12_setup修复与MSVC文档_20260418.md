## §0 关键发现（必读，不要跳过）

Claude 在 M6 验收阶段亲自跑了一次完整编译（用户机 MSVC 装在 `E:\Microsoft Visual Studio\18\BuildTools`），暴露 **3 个 bug**：

### 0.1 真编译 setup.py 漏排 `__main__.py`

opencode 的 `setup.py:get_core_extensions()` 只排除 `__init__.py`，但 `__main__.py` 也必须排除。否则：

- 编译产物会有 `core/__main__.cp312-win_amd64.pyd`
- `python -m core` 报错：`No code object available for core.__main__; 'core' is a package and cannot be directly executed`
- CLI 入口完全坏

**实证**：Claude 实际编译 → 删 `__main__.pyd` 后 CLI 才恢复正常。

正确文件计数：62 个 .py = 10 个 `__init__.py` + 1 个 `__main__.py` + **51 个真编译目标**（不是 opencode summary 写的 52）。

### 0.2 wraparound=False 触发 7 处未定义行为

opencode 的 `setup.py:cythonize(...)` 设了 `boundscheck=False, wraparound=False`，但 core/ 代码 7 处用了负索引（`lst[-1]` 等），编译时报：

```
warning: core\conversation\conversation_entry_layer.py:307:35: the result of using negative indices inside of code sections marked as 'wraparound=False' is undefined
warning: core\conversation\undo_manager.py:362:41
warning: core\inspiration\constraint_library.py:90:24
warning: core\inspiration\structural_analyzer.py:190:38 / 190:68 / 192:38 / 192:68
```

虽然当前 pytest 通过没崩，但**未定义行为是定时炸弹**。本项目不是性能瓶颈，安全优先。`wraparound=True / boundscheck=True` 是 Cython 默认，性能差异 <1%。

### 0.3 setup.py 用了 MSVC 不识别的 `-O2`

opencode 的 `setup.py:38` 写了 `extra_compile_args=["-O2"]`，这是 gcc 写法。MSVC 用 `/O2`。

但实测当前编译没报错（MSVC 把不识别的参数当 warning 跳过了），属于**沉默失败**——`-O2` 实际未生效。要么删，要么改成跨平台正确写法。

### 0.4 RELEASE.md 没说 MSVC 必须先 call vcvarsall.bat

实测：直接 `python build.py` 在没 call vcvarsall.bat 的环境下报：
```
error: Microsoft Visual C++ 14.0 or greater is required.
```

正确流程是先 `call vcvarsall.bat x64` 再 `python build.py`。RELEASE.md 没写这一步。

## §1 N12-A · 修 setup.py 排除 `__main__.py` + 改回安全默认 + 删 -O2

### 1.1 文件锚点

`setup.py`，第 20-55 行（`get_core_extensions()` 函数 + `cythonize` 调用）。

### 1.2 当前代码

```python
def get_core_extensions():
    """
    收集 core/ 目录下需要编译的 Python 文件
    排除所有 __init__.py
    """
    core_dir = Path("core")
    extensions = []

    for py_file in core_dir.rglob("*.py"):
        # 排除 __init__.py
        if py_file.name == "__init__.py":
            continue

        # 转换路径格式
        module_path = str(py_file).replace("/", ".").replace("\\", ".").replace(".py", "")

        extensions.append(
            Extension(
                name=module_path,
                sources=[str(py_file)],
                extra_compile_args=["-O2"],  # 优化
            )
        )

    return extensions

# 编译配置
ext_modules = cythonize(
    get_core_extensions(),
    compiler_directives={
        "language_level": "3",  # Python 3
        "boundscheck": False,   # 禁用边界检查（性能优化）
        "wraparound": False,    # 禁用负索引检查
    },
    exclude=["**/__init__.py"],
)
```

### 1.3 替换为（精确块，整段替换）

```python
def get_core_extensions():
    """
    收集 core/ 目录下需要编译的 Python 文件
    排除所有 __init__.py 和 __main__.py（python -m core 入口必须保留 .py 源）
    """
    core_dir = Path("core")
    extensions = []

    for py_file in core_dir.rglob("*.py"):
        # 排除 __init__.py（Cython 编译会破坏包识别）
        if py_file.name == "__init__.py":
            continue
        # 排除 __main__.py（python -m core 机制要求 .py 源，
        # 编成 .pyd 后报 "No code object available for core.__main__"）
        # [N12 2026-04-18] Claude 真编译验证发现：__main__.pyd 会让 CLI 崩
        if py_file.name == "__main__.py":
            continue

        # 转换路径格式
        module_path = str(py_file).replace("/", ".").replace("\\", ".").replace(".py", "")

        extensions.append(
            Extension(
                name=module_path,
                sources=[str(py_file)],
                # [N12 2026-04-18] 删除 extra_compile_args=["-O2"]：
                # MSVC 不识别该 GCC 风格参数（沉默失败），跨平台编译反而引入混淆。
                # MSVC 默认已开启合理优化级别，无需手工指定。
            )
        )

    return extensions

# 编译配置
ext_modules = cythonize(
    get_core_extensions(),
    compiler_directives={
        "language_level": "3",
        # [N12 2026-04-18] boundscheck/wraparound 改回 Cython 默认 True：
        # 1. core/ 7 处使用负索引（lst[-1] 等），wraparound=False 时未定义行为
        # 2. 本项目非性能瓶颈，安全优先；性能差异 <1%
        "boundscheck": True,
        "wraparound": True,
        "binding": True,         # 保留函数签名 introspection（方便调试）
        "embedsignature": True,  # docstring 含签名（IDE 提示友好）
    },
    exclude=["**/__init__.py", "**/__main__.py"],  # 双重保险
    nthreads=4,                  # 并行编译加速
)
```

```python
def get_compile_files():
    """获取需要编译的文件列表"""
    core_dir = Path("core")
    py_files = list(core_dir.rglob("*.py"))
    init_files = list(core_dir.rglob("__init__.py"))
    return [f for f in py_files if f not in init_files]
```

### 2.3 替换为

```python
def get_compile_files():
    """
    获取需要编译的文件列表
    [N12 2026-04-18] 排除 __init__.py 和 __main__.py（保持与 setup.py 一致）
    """
    core_dir = Path("core")
    py_files = list(core_dir.rglob("*.py"))
    excluded = {"__init__.py", "__main__.py"}
    return [f for f in py_files if f.name not in excluded]
```

```python
def check_msvc():
    """
    [N12 2026-04-18] 检测 MSVC 环境是否激活
    Windows 编译 .pyd 必须先 call vcvarsall.bat x64
    没激活时 distutils 会报 "Microsoft Visual C++ 14.0 or greater is required"
    """
    if sys.platform != "win32":
        return  # 非 Windows 跳过
    # 简单检测：环境变量 INCLUDE 含 MSVC 路径标志
    import os
    include = os.environ.get("INCLUDE", "")
    if "MSVC" not in include and "Visual Studio" not in include:
        print("[ERROR] MSVC 环境未激活")
        print("        请先在 Developer Command Prompt 中跑，或手动:")
        print('        cmd /c \'call "<VS路径>\\VC\\Auxiliary\\Build\\vcvarsall.bat" x64 && python build.py\'')
        print("        典型 VS 路径示例:")
        print('          E:\\Microsoft Visual Studio\\18\\BuildTools')
        print('          C:\\Program Files\\Microsoft Visual Studio\\2022\\Community')
        sys.exit(2)


def build():
    """执行 Cython 编译"""
    check_msvc()  # [N12] 编译前检测 MSVC

    compile_files = get_compile_files()
    print(f"[BUILD] 开始编译 {len(compile_files)} 个文件...")

    # 执行 setup.py
    result = subprocess.run(
        [sys.executable, "setup.py", "build_ext", "--inplace"],
        capture_output=True,
        text=True,
    )
```
