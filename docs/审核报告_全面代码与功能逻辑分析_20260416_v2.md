## 2. 系统架构概览

### 2.1 核心入口链路

```
用户输入
  └─ ConversationEntryLayer.chat()                  [conversation_entry_layer.py]
       ├─ IntentClassifier.classify()               [intent_classifier.py]
       │    └─ 返回 IntentResult(intent_name, category, entities)
       ├─ _execute_intent(intent_result)            [line 344-374]
       │    ├─ if category == SETTING_UPDATE → _execute_setting_update()  ⚠️ 死分支
       │    ├─ if category == WORKFLOW_CONTROL → _execute_workflow_control() ⚠️ 死分支
       │    ├─ if category == QUERY → _execute_query()                    ⚠️ 空壳
       │    ├─ if category == TRACKING → _execute_tracking()              ✅ 工作
       │    └─ else → IntentRouter().route()                               ⚠️ 新实例/无处理器
       └─ 返回 ConversationResponse
```

### 2.2 IntentCategory 分配现状

```python
# intent_classifier.py 中各 Intent 的实际 category 赋值：

# CHARACTER 类（add_character, update_character, delete_character）
→ IntentCategory.CHARACTER         # 无对应 _execute_xxx 分支！

# FACTION 类（add_faction, update_faction）
→ IntentCategory.FACTION           # 无对应 _execute_xxx 分支！

# PLOT 类（modify_plot, add_plot_point）
→ IntentCategory.PLOT              # 无对应 _execute_xxx 分支！

# POWER 类（add_power_system）
→ IntentCategory.POWER             # 无对应 _execute_xxx 分支！

# TIMELINE 类（add_timeline_event）
→ IntentCategory.TIMELINE          # 无对应 _execute_xxx 分支！

# WORKFLOW 类（start_chapter）
→ IntentCategory.WORKFLOW          # 无对应 _execute_xxx 分支！

# TRACKING 类（update_hook_status 等）
→ IntentCategory.TRACKING          # ✅ 有对应 _execute_tracking 分支

# 以下两个 category 在 _execute_intent 中有分支，但没有任何 Intent 会产生它们：
SETTING_UPDATE  →  _execute_setting_update()    # ← 死代码
WORKFLOW_CONTROL → _execute_workflow_control()  # ← 死代码（I17 大纲注入在此）
```

## 4. 严重缺陷（Critical）

### C1：IntentCategory 路由完全断层

**位置：** `core/conversation/intent_classifier.py` + `core/conversation/conversation_entry_layer.py:344-374`

**描述：**

`_execute_intent()` 方法基于 `IntentCategory` 做路由分支，但 `intent_classifier.py` 中所有 Intent 的实际 category 赋值为：`CHARACTER`, `FACTION`, `PLOT`, `POWER`, `TIMELINE`, `WORKFLOW`。

**没有任何 Intent 被赋值 `SETTING_UPDATE` 或 `WORKFLOW_CONTROL`。**

因此：
- `_execute_setting_update()` — 死代码
- `_execute_workflow_control()` — 死代码（I17 大纲注入在此）
- 所有写操作类 Intent 落入 `else → IntentRouter`，IntentRouter 对这些 category 无处理器

**影响：** 100% 的设定写操作静默失败，返回"处理成功"但实际什么都没做。

**修复方案（P0，预计 30 分钟）：**

```python
# intent_classifier.py 中，将以下 Intent 的 category 改为正确值：
IntentDefinition("add_character",      IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("update_character",   IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("delete_character",   IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("add_faction",        IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("update_faction",     IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("modify_plot",        IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("add_plot_point",     IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("add_power_system",   IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("add_timeline_event", IntentCategory.SETTING_UPDATE, ...)
IntentDefinition("start_chapter",      IntentCategory.WORKFLOW_CONTROL, ...)
```

---

### C2：IntentRouter 每次调用创建新实例

**位置：** `core/conversation/conversation_entry_layer.py:370`

**描述：**

```python
# 当前代码（有问题）：
return IntentRouter().route(intent_result, user_input, context)

# 应该是：
return self._intent_router.route(intent_result, user_input, context)
```

`IntentRouter` 内部通过 `pending_technique` 和 `pending_criteria` 维护多轮对话状态，每次新实例导致状态丢失。

**影响：** 所有需要"提取→确认"两步操作的流程完全失效。

**修复方案（P0，10 分钟）：**
```python
# __init__ 中：
self._intent_router = IntentRouter()

# _execute_intent 中：
return self._intent_router.route(intent_result, user_input, context)
```
