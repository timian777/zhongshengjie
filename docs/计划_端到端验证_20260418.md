## 0. 关键发现 / 与前 4 个 M 的本质不同

| 维度 | M2-α/β、M3、M4 | M5（本计划）|
|------|---------------|-----------|
| 性质 | 改代码 + 回归测试 | **跑链路 + 录证据**，原则上不改代码 |
| 成功定义 | pytest pass + grep 兜底 | 全程无 exception / fallback / 空响应；Qdrant 三集合可观察增量 |
| 失败处理 | 在本 M 内修复 | **回退到对应 M 修，不在 M5 自修**（详见 §6 失败回退矩阵）|
| 副作用 | 仅工作树 | 真实写 Qdrant、真实改 `总大纲.md` 局部、真实生成正文 → **必须先录前置快照便于回滚** |
| 成本 | 计算时间 | 5 写手 + 鉴赏师走真实 LLM API → **预算应在用户许可下进行**（见 §1.4）|

### 关键事实 1：入口已定

- 作家工作流：`python -m core create --scene <场景类型> --chapter <章节名>`（详见 [core/cli.py:285](core/cli.py:285)）
- 创作 API：`modules/creation.create_creation_api()` → `api.create_scene(scene_type=..., chapter=..., outline=...)`
- 灵感引擎分发：`core.inspiration.workflow_bridge.phase1_dispatch`
- 对话入口（用于查询/设定更新）：`core.conversation.conversation_entry_layer.ConversationEntryLayer.process(user_input)`

### 关键事实 2：路线图列的"novel_plot_v1"集合可能不存在

路线图 §2 提"`novel_plot_v1` Collection 增量"，但 [core/config_loader.py:47-51](core/config_loader.py:47) 仅配 3 个 collection：
- `novel_settings → novel_settings_v2`
- `writing_techniques → writing_techniques_v2`
- `case_library → case_library_v2`

所以 M5 可观察的增量是这 **3 个 v2 集合**，不是 `novel_plot_v1`。如发现 `novel_plot_v1` 真存在请记录但不强制要求其增量。

### 关键事实 3：`总大纲.md` PATCH 是另一条独立链路

通过 `core/change_detector/sync_manager_adapter.py` 触发——它依赖 worldview_generator（M2-β 已永久 Mock 化）。所以 M5 应观察的是"PATCH 落盘 → adapter 检测到 → Mock generator 被调用且不抛异常"，**不要求** PATCH 引发任何向量库实际变更（Mock 不写库是预期行为）。

### 关键事实 4：M5 不是一次性"全场跑通"，是 4 个独立小场景

把端到端拆成 4 个**独立可观察**的子链路，便于失败定位：
- **链路 A**：对话入口分类（intent_classifier 真跑） → `_execute_query` 走 `query_character` → 命中 novel_settings_v2
- **链路 B**：单场景作家工作流（最小迭代） → 5 写手生成 → 鉴赏师评分 → 不要求反馈回写（避免污染 case_library_v2）
- **链路 C**：`总大纲.md` 局部 PATCH → change_detector 检测 → adapter Mock 调用不抛异常
- **链路 D**：知识库 sync `--stats`（不实际同步，只读 collection 状态）

每条链路独立验收，**任一失败回退到对应 M**（§6 矩阵）。

## 6. 失败回退矩阵（**重要**）

| 链路失败 | 回退到哪个 M | 修复方向 |
|---|---|---|
| **A1 失败**：`_execute_query` 抛异常 / 返回空 message | M3 | 检查 [core/conversation/conversation_entry_layer.py:528+](core/conversation/conversation_entry_layer.py:528) 5 分支逻辑 |

| **B 失败**：作家工作流抛异常 | **不简单回退**，开 N1 调试任务 | 看 traceback 第一行所属模块；可能涉及 `modules/creation` / `core/inspiration` / 单写手 agent |
| **C 失败**：`_get_worldview_generator` 不返回 `_MockWorldviewGenerator` | M2-β | 检查 [core/change_detector/sync_manager_adapter.py](core/change_detector/sync_manager_adapter.py) 的 `_get_worldview_generator` 是否仍是 M2-β 简化版 |
| **C 失败**：`sync_from_outline` 抛 AttributeError | M2-β | Mock 类应已有此方法（[core/change_detector/sync_manager_adapter.py](core/change_detector/sync_manager_adapter.py) `_MockWorldviewGenerator`）|
| **D 失败**：连不上 Qdrant | M5 §1.1 | 不在产品代码范围，环境问题 |
| **D 失败**：collection 名不匹配（仍找 `case_library` 而非 `_v2`）| M3 | sync_manager.CASE_COLLECTION 修正可能未生效 |

**回退原则**：opencode **不在 M5 修代码**，把失败 trace 完整贴到 `docs/m5_evidence/failure_<link>.md`，停下让用户决定回到哪个 M 重开。
