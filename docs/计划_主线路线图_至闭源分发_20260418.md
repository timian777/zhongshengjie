## 3. opencode 的接力规则

每完成一个里程碑，opencode 必须按以下顺序操作：

```
1. 跑该里程碑子计划末尾的"完成验收"命令，确保产物达标
2. git add + git commit（commit message 格式见 4.1）
3. 回到本文件（docs/计划_主线路线图_至闭源分发_20260418.md）
4. 找到刚完成的里程碑那一行，把"状态"列从 ⏳ 改为 ✅，并在子计划路径列追加 commit hash，例如：
   ✅ 已完成 (commit abc12345)
5. 把"当前指针"那行（§1 末尾）更新为下一个里程碑编号
6. 再 commit 一次，message: "docs(roadmap): mark M{n} complete, advance to M{n+1}"
7. 输出一句话：「M{n} 已完成，等待 Claude 编写 M{n+1} 子计划」
8. 停下来，不要主动开始下一个 M
```

**关键**：opencode **不要**自己写下一个 M 的子计划——子计划由 Claude 写，opencode 只执行。

### 3.1 commit message 格式

每个子计划完成的 commit 必须用：
```
<type>(<scope>): <里程碑编号> · <一句话>

详见 docs/计划_主线路线图_至闭源分发_20260418.md
```

例：
```
docs(skill): M1 · migrate inspiration workflow to novel-inspiration-ingest skill

详见 docs/计划_主线路线图_至闭源分发_20260418.md
```
