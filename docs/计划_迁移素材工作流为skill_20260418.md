#### 5.4.1 落盘文件 → 同步动作映射表

| PATCH 目标文件类型 | 落盘后必跑的同步命令 | 写入的 Qdrant collection | 备注 |
|---|---|---|---|
| `设定/*.md`（含十大势力、技术基础、社会结构等） | **先**手工或脚本更新 `.vectorstore/knowledge_graph.json` 中对应实体；**后**执行 `python -m modules.knowledge_base.sync_manager --target novel` | `novel_settings` | knowledge_graph.json 是 sync 的真实数据源，不更新它则 PATCH 不入库 |

| `总大纲.md`、`分章大纲/*.md` | **不入向量库**，仅落盘即可 | — | 大纲走 outline parser，另一条链路 |
| `素材库/**`（本工作流自身产物） | **不入向量库** | — | 素材库是证据链，不是检索源 |

> ⚠️ **设定类的"先更图谱再 sync"**是当前真实状态。若用户后续要求"PATCH 设定后自动同步图谱"，那是新需求，需另开计划，不在本工作流范围内。

   ```
   确认目标 collection 的 point count 比执行前 > 旧值

| 症状 | 排查 |
|---|---|
| `sync_manager` 报 `BGE-M3 模型路径未配置` | 设 `BGE_M3_MODEL_PATH` 或检查 `config.json`，**不要**去改 sync_manager 代码 |
| `sync_manager` 报 `知识图谱不存在` | 确认 `.vectorstore/knowledge_graph.json` 在项目根的相对路径正确 |

| 同步成功但检索不到 | 用 `python -m modules.knowledge_base.search_manager --query "<关键词>"` 确认，若仍无结果回到第 3 步检查 knowledge_graph.json 是否真的写入了 |
